# Issue tracker

Issues for this repo live in **GitHub Issues** on `rmaples3/BulkXRD`.

## How agents interact with it

- **Remote / Claude Code on the web sessions**: use the GitHub **MCP tools**
  (`mcp__github__issue_write`, `mcp__github__issue_read`, `mcp__github__list_issues`,
  `mcp__github__add_issue_comment`, etc.). The `gh` CLI is NOT available in this
  environment.
- **Local sessions** (developer laptop): the `gh` CLI is the simplest path
  (`gh issue create`, `gh issue list`, `gh issue comment`).

## Conventions

- One issue = one independently-grabbable vertical slice (see the `to-issues` skill).
- Apply exactly one category label (`bug` / `enhancement`) and one state label
  (see `triage-labels.md`) per triaged issue.
- AI-generated triage comments must begin with:
  `> *This was generated by AI during triage.*`
- Do NOT open pull requests unless explicitly asked (repo rule, see CLAUDE.md).
